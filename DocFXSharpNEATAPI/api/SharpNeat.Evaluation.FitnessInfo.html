Property Value
Type Description
double[]
Remarks

Most problem tasks will yield just a single fitness value via the PrimaryFitness property, and therefore will not use this property. This is for problem tasks that produce multiple fitness values per evaluation; in those scenarios there is still a single primary fitness provided by PrimaryFitness, but there are also one or more secondary fitness scores that are for reporting purposes only, i.e., they aren't currently used by the evolution algorithm.
