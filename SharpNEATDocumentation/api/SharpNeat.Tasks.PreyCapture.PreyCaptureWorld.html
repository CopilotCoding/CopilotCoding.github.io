Class PreyCaptureWorld

The prey capture task's grid world, as defined in:

Incremental Evolution Of Complex General Behavior, Faustino Gomez and Risto Miikkulainen (1997) http://nn.cs.utexas.edu/downloads/papers/gomez.adaptive-behavior.pdf

Encapsulates the agent's sensor and motor hardware, and the prey's simple stochastic movement.

CartesianToPolar(Int32Point, out int, out float)

Convert the given integer Cartesian coordinate to a polar coordinate.

Declaration
private static void CartesianToPolar(Int32Point p, out int radiusSqr, out float azimuth)
Parameters
Type Name Description
Int32Point p

The integer Cartesian coordinate to convert.
int radiusSqr

Returns the square of the radius, as an integer. I.e. the distance from the pole, squared.
float azimuth

Returns the azimuth; the anticlockwise angle from the polar axis.
Remarks

The radius coordinate is returned as the square of the radius. This is because normally we want to compare the radius with some threshold, and it's faster to avoid a square root operation to calculate the radius, and instead to compare squared radii.

By convention the polar axis is horizontal and to the right, and a positive azimuth represents an anticlockwise rotation. Therefore an azimuth of zero degrees represents due east (note. this same angle is never represented by +360 degrees), and an angle of +90 degrees represents due north.
